Target Audience: Web developers.
Tone: Objective, developer-focused, benefits-driven.

Style Constraints:
- Keep the summary short: exactly 2 to 3 sentences.
- Sentence 1: Clearly define what the feature is and what it does.
- Sentence 2: Explain the direct benefit to developers or users (e.g., improves performance, simplifies layout code, enables new design capabilities).
- Sentence 3 (Optional): Provide a brief usage note or mention syntax.
- Avoid marketing jargon and hype. Do not use words like "introducing", "revolutionary", "exciting", "powerful new", etc.
- When writing about features shipping in Chrome, refer to "Chrome" rather than "Chromium". Refer to the browser rendering engine as "Blink".
- Only use the term "web standard" if the feature is in a W3C Recommendation/Candidate Recommendation, a WHATWG Living Standard main branch, or an ECMA TC39 Stage 4 proposal. Otherwise, describe it as a web platform feature.
- Do not use the phrase "part of the web platform" if the feature is only supported in Chromium-based browsers.
- Do not use terms like "BYOAI", "BYOM", "On-device AI", or "In-browser AI". Use "built-in AI" or "Web AI" instead.

Information Filtering (Distillation):
- Analyze the input Raw Summary for internal-facing, administrative, or non-developer information and REMOVE it.
- Specifically, strip out:
  * Enterprise policy details or policy names (e.g., GenAILocalFoundationalModelSettings, BuiltInAIAPIsEnabled). These are for enterprise admins, not general web developers.
  * Chronological version logs or release history (e.g., "Chrome M139 and earlier only supported English, M140 added support for..."). Focus only on what the feature is and does *now*.
  * Browser extension shipping history or internal bug tracker IDs (e.g., crbug.com).

Technical Enrichment & Research:
- If the input Raw Summary is extremely brief, lacks technical depth, or does not mention the specific APIs/properties being introduced, you MUST use your tools to research the feature.

  * The exact names of new CSS properties, JS methods, or HTML attributes being introduced (e.g., `column-rule-inset`, `row-rule-visibility-items`).
  * The progressive-enhancement story (e.g., what happens in unsupported browsers, how to detect support).
- Incorporate these specific technical details into your summary to make it highly informative and actionable for developers.
